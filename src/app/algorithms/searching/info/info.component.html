Searching

  • If you know a solution is not far from the root of the tree? BFS, starts searching the nodes closest to the parent first.
  • If the tree is very deep and solutions are rare? BFS > DFS, dfs can take a really long time if the solution is more towards the right of the tree
  • If the tree is very wide DFS, bfs will need too much memory in each level
  • If the solutions are frequent but located deep in the tree? DFS, should find a solution quicker as its deeper and the solutions are frequent
  • Determine whether a path exists between two nodes? DFS, thats what DFS is all aboot!
  • Finding the shortest path? BFS
